Claude Code // Computer Use

Como ativar o Computer Use no Claude Code CLI

No Claude Code de terminal, o Computer Use é ativado como um servidor MCP nativo chamado computer-use. É ele que permite ao Claude abrir apps, clicar, digitar e enxergar sua tela no macOS.

00

Antes de tudo: CLI e Desktop não ativam do mesmo jeito

O erro principal da versão anterior era misturar o fluxo do Claude Desktop app com o do Claude Code CLI. Para o guia que você vai usar na automação, o caminho correto é o do terminal.

Claude Code CLI

No terminal, o Computer Use é habilitado no menu /mcp como o servidor integrado computer-use.

claude /mcp Enable computer-use

Claude Desktop app

  • Lá, a ativação é por toggle em Settings.
  • Esse é um fluxo diferente, documentado separadamente.
  • Para este guia, o que vale é o processo do CLI.
  • Se alguém seguir o fluxo do app desktop no terminal, vai errar.
01

Pré-requisitos reais

Segundo a documentação oficial do Claude Code, o servidor computer-use só aparece em setups elegíveis.

1. Claude Desktop instalado

Baixe ou atualize em claude.com/download.

2. macOS + versão mínima

O Computer Use no CLI está disponível só no macOS e exige Claude Code v2.1.85+.

3. Sessão e autenticação corretas

Você precisa estar numa sessão interativa e autenticado via claude.ai, não só por Bedrock, Vertex ou Foundry.

Resumo rápido: macOS, Pro ou Max, Claude Code 2.1.85+, sessão interativa e login via claude.ai. Sem isso, o computer-use nem aparece no /mcp.

02

Passo a passo para ativar

Esse é o fluxo oficial da Anthropic para ativar o recurso no Claude Code CLI.

  1. Abra o terminal e inicie o Claude Code

    Entre no seu terminal, vá para o projeto e rode claude para abrir uma sessão interativa normal.

  2. Abra o gerenciador de MCP

    Dentro da sessão, rode /mcp. É ali que ficam os servidores MCP disponíveis para o projeto atual.

  3. Encontre o servidor computer-use

    Procure por computer-use na lista. Ele aparece como um servidor MCP embutido e, por padrão, começa como disabled.

  4. Ative o computer-use

    Selecione computer-use e escolha Enable. Essa configuração persiste por projeto, então você não precisa repetir toda vez no mesmo repositório.

  5. Conceda as permissões do macOS

    Na primeira tentativa de uso, o macOS vai pedir Accessibility para clicar, digitar e rolar, e Screen Recording para Claude enxergar a tela. Depois de conceder, clique em Try again.

  6. Reinicie a sessão se o macOS pedir

    A própria documentação avisa que o macOS pode exigir reinício do Claude Code depois da permissão de Screen Recording. Se o prompt ficar reaparecendo, feche e abra uma nova sessão.

  7. Peça uma tarefa visual e aprove os apps da sessão

    A partir daí, peça algo que realmente precise de GUI. Quando Claude quiser controlar um app específico, o terminal vai mostrar um prompt para Allow for this session ou Deny.

Dica prática: se o servidor computer-use não aparecer no /mcp, quase sempre o motivo é um destes: plano errado, macOS ausente, versão antiga, sessão não interativa ou autenticação só via provedor de terceiros.

03

Como funcionam as permissões

Ativar o servidor não libera o seu Mac inteiro de uma vez. O Claude continua pedindo aprovação por app durante a sessão.

O que a sessão pede

  • Quais apps Claude quer controlar.
  • Se há permissões extras, como acesso ao clipboard.
  • Quantos outros apps serão escondidos enquanto Claude trabalha.
  • Se o app aprovado dá alcance amplo, como Finder ou System Settings.

Regras úteis de operação

  • As aprovações valem só para a sessão atual.
  • O terminal fica visível, mas é excluído dos screenshots.
  • Outros apps visíveis são escondidos enquanto Claude controla a tela.
  • Você pode parar tudo com Esc ou Ctrl+C.
04

Como pedir do jeito certo

O ganho aqui não é só ligar o servidor. É formular pedidos que façam o Claude operar de forma previsível e segura.

Prompts que funcionam melhor

  • "Abra o app X, exporte o relatório Y e salve na pasta Z."
  • "Entre no dashboard interno, filtre por março e me traga os números principais."
  • "Abra meu projeto, rode os testes e me diga onde quebrou."
  • "Navegue nessa tela e complete só os campos A, B e C."

Evite começar assim

  • "Resolve tudo aí."
  • "Mexe no que precisar e vê no que dá."
  • Pedidos sem app alvo, resultado esperado ou limite claro de ação.
  • Tarefas sensíveis envolvendo finanças, saúde, dados legais ou contas críticas.
05

Permissões e segurança

O Computer Use roda no seu desktop real, não em um sandbox isolado. Então vale usar como ferramenta poderosa, não como automação cega.

Comece pelos apps confiáveis

Feche janelas sensíveis antes de usar e aprove app por app com bastante critério.

A rota mais precisa vem antes

Se houver MCP, Bash ou Chrome extension, Claude tenta isso antes de usar a tela.

Use supervisão humana

A própria Anthropic recomenda cautela com prompt injection e tarefas de alto risco.

06

Limites atuais

A documentação do CLI deixa claro onde o recurso funciona e onde ele ainda não entra.

Uma sessão por vez

Existe um lock global da máquina. Se outra sessão estiver usando a tela, a nova falha.

Só no modo interativo

O recurso não funciona no modo não interativo com a flag -p.

Só no macOS

Linux e Windows ainda não entram no fluxo do Computer Use do CLI.

07

Diferença do Desktop app

Se você viu tutoriais conflitantes, esta é a diferença que resolve a confusão.

No Claude Code CLI

  • Ativação em /mcp.
  • Servidor embutido computer-use.
  • Fluxo oficial para terminal e automações orientadas ao CLI.

No Desktop app

  • Ativação por toggle em Settings.
  • É a superfície visual do app, não o fluxo do terminal.
  • Não é o caminho certo para um tutorial de ativação do CLI.
08

Resumo rápido

Se você quiser decidir em 20 segundos se já dá para ligar o recurso, use essa tabela.
