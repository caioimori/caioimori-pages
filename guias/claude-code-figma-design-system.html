00

O que muda com Figma + Claude Code

Antes voce exportava assets, copiava hex codes e recriava espacamentos no olho. Agora o Claude Code le o canvas diretamente.

Leitura direta

Claude Code acessa a arvore de nodes do Figma via MCP. Ve hierarquia, propriedades, tokens e variantes sem voce copiar nada.

Contexto visual

Alem dos dados estruturados, o MCP captura screenshots dos frames para que o Claude interprete o layout visualmente.

Escrita no canvas

O fluxo e bidirecional. Voce pode pedir ao Claude para criar componentes, diagramas e prototipos direto no Figma.

01

Setup: conectando Figma ao Claude Code

O Figma MCP e oficial da Anthropic e ja vem disponivel no Claude Code. Basta autorizar.

  1. Abra o Claude Code no seu projeto

    Navegue ate a pasta do projeto e rode claude. O MCP do Figma ja esta registrado automaticamente.

  2. Cole a URL do Figma no prompt

    Copie a URL do frame ou componente no Figma (ex: figma.com/design/ABC123/...) e cole direto no chat. O Claude detecta automaticamente que e uma URL do Figma.

  3. Autorize o acesso na primeira vez

    Na primeira chamada, o Claude vai pedir autorizacao para acessar o Figma. Aceite a permissao — ela vale para toda a sessao.

  4. Pronto. O Claude ja le o canvas.

    A partir daqui, voce pode pedir para extrair tokens, gerar componentes, comparar design com codigo existente e muito mais.

Importante: voce nao precisa instalar nenhum plugin, pacote npm ou configurar servidor MCP manualmente. O Figma MCP e nativo do Claude Code.

02

Extraindo o design system

O primeiro uso pratico: transformar o que esta no Figma em tokens, variaveis CSS e componentes reais do seu projeto.

O que o MCP extrai

  • Cores — paleta completa com nomes de variaveis do Figma e valores hex/rgba.
  • Tipografia — font families, sizes, weights, line-heights e letter-spacings.
  • Espacamento — padding, gap e margin dos auto-layouts.
  • Border radius — valores exatos de cada componente.
  • Efeitos — shadows, blurs e opacidades aplicadas.

Como pedir

  • "Extrai o design system desse arquivo do Figma" + URL
  • "Gera as CSS variables baseado nesse frame"
  • "Quais sao os tokens de cor e tipografia desse design?"
  • "Cria um arquivo de theme baseado nesse Figma"
  • "Compara esse design com meu CSS atual e mostra as diferencas"

Exemplo de resultado

Ao colar uma URL e pedir para extrair tokens, o Claude gera algo como:

:root { /* Colors — extraido do Figma */ --color-primary: #1a1a1a; --color-secondary: #6e6e6e; --color-accent: #2563eb; --color-surface: #f7f7f7; --color-border: #e5e5e5; /* Typography */ --font-heading: 'Inter', sans-serif; --font-body: 'Inter', sans-serif; --font-size-hero: 3rem; --font-size-h2: 1.75rem; --font-size-body: 1rem; --line-height-tight: 1.1; --line-height-normal: 1.6; /* Spacing */ --space-xs: 4px; --space-sm: 8px; --space-md: 16px; --space-lg: 24px; --space-xl: 48px; /* Radius */ --radius-sm: 8px; --radius-md: 16px; --radius-full: 100px; }
03

Design-to-code: do frame ao componente

O fluxo principal. Voce aponta um frame no Figma e o Claude gera o componente pronto, adaptado ao stack do seu projeto.

  1. Selecione o frame no Figma

    Clique no componente ou frame que quer transformar em codigo. Copie a URL com o node selecionado (o ?node-id= aparece automaticamente).

  2. Cole no Claude Code com instrucao

    Exemplo: "Implementa esse card como componente React seguindo meu design system" + URL. O Claude usa get_design_context para ler o frame.

  3. Claude analisa a arvore de nodes

    O MCP retorna a estrutura hierarquica do frame (layers, auto-layout, fills, strokes, text nodes), alem de um screenshot para referencia visual.

  4. Codigo gerado e adaptado ao projeto

    O Claude nao gera codigo generico. Ele verifica seu codebase, encontra componentes existentes, tokens ja definidos e padroes de layout — e reutiliza tudo.

Dica: quanto mais organizado o Figma (auto-layout, nomes claros, variaveis de design), melhor o codigo gerado. Frames com posicionamento absoluto e cores hardcoded produzem resultados mais genericos.

04

Code Connect: mapeando componentes

O recurso que torna o fluxo repetitivo muito mais eficiente. Voce vincula um componente do Figma a um componente do seu codebase — uma vez — e o Claude sempre usa o certo.

O que e Code Connect

E um mapeamento persistente entre componentes do Figma e componentes do codigo. Quando o Claude encontra um componente mapeado no design, ele usa diretamente o import do seu projeto em vez de recriar do zero.

Exemplo: o componente "Button/Primary" no Figma fica vinculado a <Button variant="primary" /> do seu codebase.

Como configurar

  • Peca ao Claude: "Configura Code Connect para esse arquivo do Figma"
  • Ele vai sugerir mapeamentos entre componentes do Figma e do codigo
  • Voce confirma, ajusta ou adiciona mapeamentos manualmente
  • Os mapeamentos ficam salvos e sao usados em todas as proximas geracoes

Comandos uteis

Ver mapeamentos "Mostra os Code Connect mappings atuais"
Adicionar mapping "Mapeia o componente X do Figma para o componente Y do meu codigo"
Sugestoes automaticas "Sugere Code Connect mappings para esse arquivo do Figma"
Gerar com mappings "Implementa essa tela usando os Code Connect mappings" + URL
05

Code-to-canvas: escrevendo de volta no Figma

O fluxo inverso. Voce pede ao Claude para criar ou modificar elementos diretamente no canvas do Figma.

Criar arquivos

O Claude pode criar novos arquivos no Figma a partir de descricoes textuais ou baseado em codigo existente.

Gerar diagramas

Diagramas de arquitetura, fluxos de usuario e wireframes criados direto no FigJam a partir de instrucoes no prompt.

Design system rules

Definir regras do design system no Figma que o Claude segue automaticamente em geracoes futuras.

Exemplos praticos de code-to-canvas

  • "Cria um diagrama de arquitetura do meu projeto no FigJam"
  • "Gera um wireframe dessa feature no Figma baseado na minha descricao"
  • "Cria as design system rules no Figma baseado no meu CSS atual"
  • "Atualiza os tokens de cor no Figma para refletir as mudancas que fiz no codigo"
  • "Cria um novo arquivo no Figma com os componentes que implementei"
06

Figma Console: controle total via codigo

Para quem quer ir alem do MCP oficial, o Figma Console MCP permite executar codigo diretamente no Figma — criar componentes, manipular layers, instanciar da biblioteca e mais.

O que voce pode fazer

  • Criar componentes — frames, textos, shapes e component sets programaticamente.
  • Instanciar da biblioteca — buscar componentes existentes e criar instancias.
  • Manipular propriedades — fills, strokes, auto-layout, constraints, tudo via codigo.
  • Capturar screenshots — validar visualmente apos cada modificacao.
  • Gerenciar variaveis — criar, editar e deletar design tokens direto no Figma.

Quando usar

  • Batch operations — criar 20 variantes de um componente de uma vez.
  • Design system audit — verificar consistencia de tokens e componentes.
  • Prototipagem rapida — gerar telas inteiras a partir de descricoes.
  • Sincronizacao — manter tokens do Figma alinhados com o codigo.

Prerequisito: o Figma Console MCP requer o plugin do Figma Console instalado no seu Figma. Diferente do MCP oficial, ele executa codigo diretamente na API do plugin.

07

Workflow completo na pratica

O ciclo ideal quando voce usa Figma + Claude Code juntos no dia a dia.

  1. Designer finaliza o frame no Figma

    Com auto-layout, variaveis de design, nomes claros e componentes da biblioteca. Quanto mais estruturado, melhor o output.

  2. Dev cola a URL no Claude Code

    "Implementa essa tela" + URL. O Claude le o design, identifica componentes existentes via Code Connect e gera o codigo adaptado ao projeto.

  3. Claude gera e voce revisa

    O codigo ja usa seus tokens, seus componentes e seus padroes. Voce revisa, ajusta detalhes e comita.

  4. Mudancas no codigo refletem no Figma

    Se voce mudou tokens, cores ou componentes no codigo, pode pedir ao Claude para sincronizar de volta no Figma.

  5. Design system rules garantem consistencia

    Defina regras no Figma que o Claude segue automaticamente — nomes de classes, padroes de componentes, grid systems.

08

Dicas para fidelidade maxima

O que fazer no Figma para que o Claude gere codigo mais fiel ao design.

Faca isso

  • Use auto-layout em tudo — ele vira flexbox/grid no codigo.
  • Nomeie layers com clareza — "hero-section", "card-title", nao "Frame 427".
  • Use variaveis de design para cores e espacamento — viram CSS variables.
  • Organize componentes na biblioteca — o Code Connect funciona melhor.
  • Adicione annotations em frames complexos — o Claude le e segue as instrucoes.
  • Use fill container em vez de hug contents quando fizer sentido.

Evite isso

  • Posicionamento absoluto sem necessidade — gera codigo fragil.
  • Cores hardcoded fora de variaveis — o Claude nao tem como nomear.
  • Layers com nomes genericos (Frame 1, Group 2) — perdem semantica.
  • Fontes que nao estao no projeto — o Claude vai referenciar e nao vai funcionar.
  • Efeitos complexos sem fallback — nem tudo traduz 1:1 para CSS.
  • Componentes soltos fora de frames — perde contexto de layout.
