Auto-Fix no Claude Code

Configure hooks que formatam, lintam e verificam seu código automaticamente toda vez que o Claude edita um arquivo. Zero intervenção manual.

Este guia é um conteúdo SINAPSE — aprenda IA aplicada com clareza e execução →

O que é o auto-fix?

O Claude Code tem um sistema de hooks — comandos que rodam automaticamente em momentos específicos do ciclo de vida. Você configura uma vez e, a partir daí, toda vez que o Claude edita um arquivo, o hook roda o formatter e o linter sem você precisar pedir.

O ciclo

Claude edita arquivo → hook roda Prettier/ESLint → se tiver erro, Claude vê e corrige → hook roda de novo → até passar tudo. É um loop auto-corretivo.

1. Instalar o Claude Code

Se você ainda não tem o Claude Code instalado:

# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash

# Windows (PowerShell)

Verifique a instalação:

claude --version
claude doctor
Pré-requisitos

Windows precisa do Git for Windows. Qualquer OS precisa de 4GB+ de RAM e um terminal com Bash, Zsh ou PowerShell.

2. Como os hooks funcionam

Hooks são comandos de shell determinísticos que rodam em momentos específicos. Não dependem da IA decidir rodar — eles disparam automaticamente quando a condição é atendida.

PostToolUse

Roda depois que o Claude edita/cria um arquivo. É aqui que você coloca formatters e linters.

PreToolUse

Roda antes de uma ação. Pode bloquear a execução (exit code 2). Útil pra proteger arquivos.

Stop

Roda quando o Claude termina de responder. Se falhar, o Claude continua trabalhando até resolver.

Matcher

Filtra qual ferramenta dispara o hook. Edit|Write = só quando editar ou criar arquivo.

Exit Code Significado
0 Sucesso — ação permitida
2 Bloqueio — ação impedida (PreToolUse)
outro Erro — Claude vê o stderr e tenta corrigir

3. Onde configurar

Hooks ficam no arquivo settings.json. Existem três escopos:

Escopo Caminho Compartilhável
Global ~/.claude/settings.json Não — só seu
Projeto .claude/settings.json Sim — commita no git
Local .claude/settings.local.json Não — gitignored

Para auto-fix que vale pro time inteiro, use o escopo de projeto (.claude/settings.json) e commite no repositório.

4. Setup básico — Prettier + ESLint

O caso de uso mais comum: formatar e lintar automaticamente depois de cada edição. Crie ou edite o arquivo .claude/settings.json no seu projeto:

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'FILE=$(jq -r \".tool_input.file_path\") && npx prettier --write \"$FILE\" && npx eslint --fix \"$FILE\"'",
            "timeout": 120
          }
        ]
      }
    ]
  }
}

O que acontece:

  1. Claude edita ou cria um arquivo (dispara PostToolUse)
  2. O hook recebe o path do arquivo via JSON no stdin
  3. jq extrai o file_path do JSON
  4. Prettier formata o arquivo in-place
  5. ESLint corrige o que conseguir automaticamente
Dependência

Você precisa do jq instalado no sistema. No macOS: brew install jq. No Ubuntu: sudo apt install jq. No Windows com Git Bash, jq pode ser baixado como binário standalone.

5. Type-check como gate de saída

Além do auto-fix por arquivo, você pode adicionar um Stop hook que verifica o projeto inteiro antes do Claude parar. Se falhar, o Claude continua trabalhando até resolver.

{
  "hooks": {
    "PostToolUse": [
      // ... hooks de formatação acima
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "npx tsc --noEmit",
            "timeout": 60
          }
        ]
      }
    ]
  }
}

tsc --noEmit verifica tipos sem gerar output. Se sair com erro, o Claude vê as mensagens de erro no stderr e automaticamente tenta corrigir os problemas de tipo — e o hook roda de novo até passar.

6. Configs por linguagem

O comando do hook muda conforme a stack. Aqui estão os mais comuns:

Python — Black + Ruff

"command": "FILE=$(jq -r '.tool_input.file_path') && [[ \"$FILE\" == *.py ]] && black \"$FILE\" && ruff check --fix \"$FILE\" || true"

Go — gofmt + golangci-lint

"command": "FILE=$(jq -r '.tool_input.file_path') && [[ \"$FILE\" == *.go ]] && gofmt -w \"$FILE\" && golangci-lint run --fix \"$FILE\" || true"

Rust — rustfmt + clippy

"command": "FILE=$(jq -r '.tool_input.file_path') && [[ \"$FILE\" == *.rs ]] && rustfmt \"$FILE\" && cargo clippy --fix --allow-dirty || true"
O padrão

Todos seguem a mesma estrutura: jq extrai o path → checa extensão → roda formatter → roda linter com --fix. O || true evita que o hook falhe se o arquivo não for da linguagem certa.

7. Proteger arquivos sensíveis

Use um PreToolUse hook para impedir que o Claude edite arquivos que não deveria tocar:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'FILE=$(jq -r \".tool_input.file_path\") && for p in .env package-lock.json; do [[ \"$FILE\" == *\"$p\"* ]] && echo \"Blocked: $FILE\" >&2 && exit 2; done; exit 0'",
            "timeout": 10
          }
        ]
      }
    ]
  }
}

Exit code 2 = bloqueio. O Claude recebe a mensagem do stderr e ajusta a abordagem automaticamente.

8. Config completa de referência

Juntando tudo em um único .claude/settings.json para um projeto TypeScript/React:

{
  "permissions": {
    "defaultMode": "acceptEdits"
  },
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{
          "type": "command",
          "command": "bash -c 'FILE=$(jq -r \".tool_input.file_path\") && for p in .env credentials.json; do [[ \"$FILE\" == *\"$p\"* ]] && echo \"Blocked\" >&2 && exit 2; done; exit 0'",
          "timeout": 10
        }]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [{
          "type": "command",
          "command": "bash -c 'FILE=$(jq -r \".tool_input.file_path\") && [[ \"$FILE\" =~ \\.(ts|tsx|js|jsx)$ ]] || exit 0 && npx prettier --write \"$FILE\" 2>/dev/null && npx eslint --fix \"$FILE\" 2>/dev/null || true'",
          "timeout": 120
        }]
      }
    ],
    "Stop": [
      {
        "hooks": [{
          "type": "command",
          "command": "npx tsc --noEmit",
          "timeout": 60
        }]
      }
    ]
  }
}
acceptEdits

O defaultMode: "acceptEdits" faz o Claude editar arquivos sem pedir permissão. Essencial pro auto-fix funcionar sem fricção. O hook roda imediatamente após cada edição.

9. Verificar se está funcionando

  1. Checar hooks carregados

    Dentro do Claude Code, digite /hooks para ver todos os hooks ativos e confirmar que foram carregados corretamente.

  2. Testar manualmente

    Simule o input que o hook recebe:

    echo '{"tool_input":{"file_path":"src/index.ts"}}' | bash -c 'FILE=$(jq -r ".tool_input.file_path") && echo "Would format: $FILE"'
  3. Rodar o Claude e observar

    Peça pro Claude editar qualquer arquivo. Você vai ver no transcript que o hook rodou automaticamente após a edição.

10. Troubleshooting

Hook não dispara

Verifique o matcher (case-sensitive), o evento correto (PostToolUse vs PreToolUse) e a sintaxe JSON (sem trailing commas).

jq não encontrado

Instale o jq: brew install jq (macOS), apt install jq (Ubuntu) ou baixe o binário para Windows.

Erro de parse JSON

Cheque se seu ~/.zshrc ou ~/.bashrc tem echo fora de bloco interativo. Envolva com if [[ $- == *i* ]].

Hook lento demais

Aumente o timeout ou use "async": true para hooks que não precisam bloquear a execução.
