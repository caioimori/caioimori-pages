Como o /ultraplan funciona

O fluxo é assíncrono: você pede, continua trabalhando, e revisita quando quiser.

1. Você digita /ultraplan no terminal

Descreve a feature ou tarefa complexa que precisa de planejamento.

2. Claude envia para a nuvem

O trabalho pesado vai para o Opus 4.6 rodando em cloud. Seu terminal fica completamente livre.

3. O plano abre em interface web

Quando pronto, você recebe um link para uma interface tipo Google Docs, mas para código.

4. Você colabora em tempo real

Comente em linhas específicas, edite a arquitetura, peça revisão — tudo sincronizado com a IA.

5. Escolha como executar

Execute na nuvem (PR automático no GitHub) ou traga o plano para codificar localmente.

01

O fluxo antigo vs. o novo

A mudança é arquitetural no jeito de trabalhar com agentes.

Antes

  • Pede plano → terminal trava
  • Espera minutos olhando para a tela
  • Recebe texto plano no terminal
  • Executa cegamente ou volta para editar
  • Sem colaboração visual

Com /ultraplan

  • Pede plano → continua trabalhando
  • Revisita quando quiser
  • Colabora visualmente em interface web
  • Comenta em linhas específicas do plano
  • Executa com confiança após revisão
02

Por que Opus 4.6 na nuvem muda a qualidade

Rodar o planejamento na nuvem com o modelo mais potente tem impacto direto no resultado.

Modelo mais potente

Opus 4.6 rodando na nuvem significa capacidade computacional muito maior do que rodaria localmente. Planos mais completos e estruturados.

Menos refactoring depois

Estrutura melhor na fase de planejamento significa menos retrabalho na implementação. O investimento no plano se paga rápido.

Sem perda de produtividade

Como é assíncrono, você não perde tempo esperando. O agente trabalha enquanto você faz outras coisas.

03

As duas formas de executar

Depois que o plano está refinado, você escolhe como quer que ele vire código.

Execução na nuvem

  • O agente implementa o plano em cloud
  • Abre um Pull Request direto no GitHub
  • Você revisa o PR como faria normalmente
  • Ideal para features independentes

Execução local

  • Baixa o plano para sua máquina
  • Codifica localmente com controle total
  • Usa o plano como guia de implementação
  • Ideal quando precisa de contexto local
