O que é o Pencil

Entenda o conceito antes de instalar. Faz toda a diferença saber o que você está colocando no seu fluxo.

Design as Code

No Pencil, cada design é um arquivo .pen que fica no seu repositório, versionado com Git, igual ao código. Sem arquivos no Figma, sem dependência de nuvem de terceiros.

O canvas do Pencil vive dentro do VS Code, Cursor ou Claude Code. Você desenha no editor, o resultado é código.

IA nativa via MCP

Quando o Pencil está aberto, ele sobe automaticamente um servidor MCP local. Claude Code e outros assistentes de IA conseguem ler, criar e modificar designs diretamente pelo protocolo MCP.

Você faz um pedido em linguagem natural. O Claude executa as mudanças no canvas. Sem copiar e colar, sem exports manuais.

Em resumo: Pencil é o Figma que vive no seu terminal, fala com o Claude Code e versiona junto com o código. Tudo local, tudo gratuito.

01

Instalação do Pencil

Três formas de instalar. Escolha a que faz mais sentido para o seu fluxo.

Extensão do VS Code / Cursor

Recomendado se você já usa um desses editores. Busque "Pencil" nas extensões com Cmd/Ctrl + Shift + X e clique em instalar.

App desktop

Para quem prefere o Pencil como janela separada. Disponível para macOS, Windows e Linux. Baixe em pencil.dev/download.

CLI via npm

Para integração em pipelines ou uso headless. Instale com npm install -g @pencil.dev/cli.

macOS

Baixe o .dmg correto para o seu chip e arraste para a pasta Aplicativos.

# Apple Silicon (M1 / M2 / M3 / M4) pencil.dev/download/Pencil-mac-arm64.dmg # Intel pencil.dev/download/Pencil-mac-x64.dmg

Windows

Baixe o instalador .exe e execute normalmente. Não precisa de permissão de administrador.

# Intel / AMD x64 pencil.dev/download/Pencil-win-x64.exe # ARM64 — em breve

Linux

AppImage (recomendado — sem instalação)

# x64 pencil.dev/download/Pencil-linux-x86_64.AppImage # ARM pencil.dev/download/Pencil-linux-arm64.AppImage

Tarball (instalação manual)

# x64 pencil.dev/download/Pencil-linux-x64.tar.gz # ARM pencil.dev/download/Pencil-linux-arm64.tar.gz

Linux: X11 é mais estável que Wayland/Hyprland no momento.

  1. Instale o Pencil

    Via extensão do editor (recomendado) ou app desktop. Ambos funcionam com Claude Code.

  2. Crie sua conta em pencil.dev

    O cadastro é gratuito. Confirme o e-mail de ativação — sem isso, os recursos de IA não ficam disponíveis.

  3. Faça login no Pencil

    Abra o Pencil e entre com a conta criada. A autenticação é local e não exige token de API.

  4. Abra o arquivo de boas-vindas

    O Pencil cria um arquivo .pen de exemplo automaticamente. Abra para confirmar que o canvas está funcionando.

02

Integração com Claude Code

O MCP server do Pencil sobe automaticamente. Não precisa configurar nada — só confirmar que a conexão está ativa.

Como funciona

  • Pencil abre → servidor MCP local sobe automaticamente.
  • Claude Code detecta o MCP e consegue ler e modificar arquivos .pen.
  • Tudo fica na sua máquina. Nenhum arquivo de design vai para a nuvem.
  • Você mantém o controle: o Claude sugere, você aprova.

Verificar a conexão

Para confirmar que o MCP está ativo no Cursor, vá em Settings > Tools & MCP e verifique se o Pencil aparece listado.

No Claude Code CLI, rode:

claude # Dentro do Claude Code: /mcp

O servidor do Pencil deve aparecer como conectado enquanto o app estiver aberto.

  1. Garanta que o Claude Code está autenticado

    Rode claude --version para confirmar. Se precisar autenticar, rode claude e siga o fluxo de login no navegador.

  2. Abra o Pencil com um arquivo .pen

    O servidor MCP só sobe quando o Pencil está aberto e com um arquivo .pen ativo. Não precisa de configuração adicional.

  3. Acesse o prompt de IA

    Dentro do Pencil, pressione Cmd/Ctrl + K para abrir o prompt de IA e começar a interagir com o Claude.

  4. Faça seu primeiro pedido

    Tente algo simples: "Crie um botão primário com fundo azul e texto branco". O Claude executa no canvas em tempo real.

Claude Code CLI: se você usa o CLI em vez do app desktop, abra o Pencil em paralelo em outra janela do terminal ou no editor. O MCP é detectado automaticamente pela sessão ativa do Claude Code.

03

O que você pode pedir para o Claude fazer

O MCP do Pencil expõe ferramentas de design direto para o Claude. Veja o que é possível e exemplos de prompts que funcionam.

Criar elementos

"Crie um card com avatar, nome e descrição" — o Claude insere os elementos no canvas com layout, espaçamento e hierarquia.

Modificar estilos

"Mude todos os botões primários para azul" — o Claude busca os elementos por tipo e aplica as mudanças em lote.

Gerar código

"Gere o código React desse componente" — o Pencil converte o design em código pronto para usar no projeto.

Ferramentas MCP disponíveis

batch_design # Criar, mover, copiar, deletar elementos batch_get # Ler componentes e estrutura do design get_screenshot # Gerar preview e verificar resultado visual snapshot_layout # Analisar estrutura e posicionamento get_editor_state # Ver contexto e seleção atual get_variables # Ler design tokens e variáveis CSS set_variables # Atualizar tokens e sincronizar com CSS

Exemplos de prompts

  • "Crie um dashboard com sidebar e área de conteúdo principal."
  • "Adicione variantes hover e disabled ao botão primário."
  • "Analise o layout e corrija os problemas de alinhamento."
  • "Gere um sistema de tipografia com escala para headings e body."
  • "Converta esse componente em React com Tailwind."
04

Problemas comuns e soluções rápidas

A maioria dos erros tem origem na autenticação ou no Pencil não estar aberto com um arquivo ativo.

Claude Code não conecta ao MCP Verifique o login com claude e reinicie o Pencil com um arquivo .pen aberto.
Servidor MCP não aparece no editor Confirme que o Pencil está rodando. Verifique em Settings > Tools & MCP (Cursor) ou /mcp no CLI.
Recursos de IA indisponíveis E-mail de ativação não confirmado. Verifique a caixa de spam ou reinstale a extensão para reenviar.
Erros de permissão no canvas Aceite os prompts de permissão que o Pencil exibe na primeira vez. Verifique permissões da pasta do projeto.
Mudanças inesperadas no design Use prompts mais específicos. Peça para o Claude explicar o que vai fazer antes de executar.
Linux — tela em branco ou crash Tente X11 em vez de Wayland/Hyprland. Use o AppImage que não exige instalação de dependências.
