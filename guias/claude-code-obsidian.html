Segundo Cérebro com Claude Code + Obsidian

Um sistema de conhecimento pessoal que se alimenta sozinho todos os dias — sem você precisar abrir nenhum app ou organizar nada manualmente.

Este guia é um conteúdo SINAPSE — aprenda IA aplicada com clareza e execução →

A ideia

Você programa um agente de IA para entrar no seu vault do Obsidian todo dia no mesmo horário. Ele revisa tudo que você fez durante o dia, organiza notas soltas, corrige links quebrados e complementa com informação nova.

O efeito acumulado é o mais interessante: depois de algumas semanas, as notas ficam cada vez mais conectadas, o contexto fica mais rico e o agente passa a te ajudar melhor em tudo — porque ele já tem acesso a todo o histórico organizado.

O ciclo

Quanto mais o agente alimenta o vault, mais inteligente o sistema fica. É um ciclo que se retroalimenta. Tudo que você aprende, produz e consome entra no vault e se conecta com o que já existe — com zero esforço manual.

Por que Obsidian?

Obsidian é a base ideal para esse sistema por três razões:

Arquivos locais

Tudo são arquivos .md no seu computador. O Claude Code lê e escreve diretamente neles sem precisar de API nenhuma.

Links bidirecionais

Os [[wikilinks]] criam uma rede de conhecimento estilo Wikipedia. O agente usa isso pra conectar notas novas com as que já existem.

Graph View

Obsidian renderiza um grafo visual de todas as conexões. Conforme o agente alimenta, o grafo cresce e você vê o conhecimento se formando.

Portabilidade total

Não depende de nenhuma plataforma. São seus arquivos, no seu computador, sincronizáveis com qualquer serviço de nuvem.

Estrutura do vault

Organize seu vault com pastas que o agente vai usar como referência. Não precisa ser perfeito desde o início — o agente vai organizar com o tempo.

vault/
├── inbox/             # notas rápidas e capturas do dia
├── notas/             # notas processadas e organizadas
├── projetos/          # notas por projeto ativo
├── referencias/       # artigos, livros, recursos externos
├── templates/         # templates de nota (daily, meeting, etc)
├── daily/             # daily notes automáticas
└── CLAUDE.md          # instruções para o agente
Inbox é a chave

Durante o dia, jogue tudo na inbox/ sem se preocupar com organização. O agente cuida de mover, renomear e conectar cada nota no horário programado.

O arquivo CLAUDE.md

O CLAUDE.md fica na raiz do vault e funciona como as instruções permanentes do agente. Toda vez que o Claude Code abre o vault, ele lê esse arquivo primeiro. É aqui que você define o comportamento da manutenção automática.

# CLAUDE.md — Instruções do Second Brain

Você é o curador do meu vault de conhecimento pessoal
no Obsidian. Toda noite você executa uma rotina de
manutenção seguindo estas regras:

## Regras gerais
- Use [[wikilinks]] para conectar notas relacionadas
- Nunca delete notas — reorganize e conecte
- Mantenha um tom consistente e conciso
- Adicione tags relevantes no frontmatter YAML
- Preserve o conteúdo original, apenas enriqueça

## Rotina noturna (nightly maintenance)

### 1. Processar inbox
- Leia cada nota em inbox/
- Mova para a pasta apropriada (notas/, projetos/, etc)
- Renomeie com título descritivo
- Adicione frontmatter (tags, date, related)
- Crie [[links]] para notas existentes relevantes

### 2. Verificar links
- Encontre [[links]] quebrados (apontam pra nota que
  não existe)
- Corrija ou crie a nota de destino com conteúdo stub

### 3. Enriquecer notas recentes
- Revise notas modificadas nos últimos 7 dias
- Adicione conexões que faltam com notas mais antigas
- Sugira relações entre conceitos de áreas diferentes

### 4. Daily summary
- Crie uma nota em daily/ com resumo do que foi
  processado
- Liste notas movidas, links criados e conexões feitas

Esse é um ponto de partida. Personalize conforme o tipo de conteúdo que você mais captura — artigos, reuniões, ideias de projeto, aprendizados técnicos.

Como rodar o agente

O Claude Code roda direto no terminal apontando para o diretório do vault. O comando base é:

claude -p "Execute a rotina noturna de manutenção do vault conforme o CLAUDE.md" \
  --allowedTools "Read,Write,Edit,Glob,Grep" \
  /caminho/para/seu/vault

A flag -p passa o prompt direto sem abrir uma sessão interativa. O agente lê o CLAUDE.md, executa a rotina e encerra. As allowedTools garantem que ele só use operações de arquivo — sem rodar comandos arbitrários.

Dica

Rode manualmente primeiro para ver o que o agente faz. Ajuste o CLAUDE.md até o comportamento estar do jeito que você quer antes de automatizar.

Agendar para rodar todo dia

Depois de testar manualmente, agende a execução automática. Duas opções:

  1. macOS / Linux — crontab Abra o terminal e rode crontab -e. Adicione a linha:
    0 23 * * * cd /caminho/vault && claude -p "Execute a rotina noturna de manutenção" --allowedTools "Read,Write,Edit,Glob,Grep"

    Isso roda todo dia às 23h. Ajuste o horário conforme preferir.

  2. Windows — Task Scheduler Crie uma tarefa agendada que execute um script .bat:
    cd C:\Users\SeuUser\vault
    claude -p "Execute a rotina noturna de manutenção" --allowedTools "Read,Write,Edit,Glob,Grep"

    Configure o trigger para diariamente às 23:00 no Task Scheduler.

  3. Claude Code Triggers (nativo) Se você usa Claude Code com suporte a scheduled triggers:
    claude schedule create \
      --name "nightly-vault" \
      --cron "0 23 * * *" \
      --prompt "Execute a rotina noturna de manutenção do vault"

O que o agente faz na prática

Processa inbox

Pega notas jogadas às pressas durante o dia, dá títulos descritivos, move pra pasta certa e adiciona metadados.

Conecta notas

Encontra relações entre notas que você nunca faria manualmente. Um artigo sobre marketing se conecta com uma ideia de produto de 3 meses atrás.

Corrige links

Identifica [[links]] que apontam pra nada e resolve — criando a nota ou corrigindo o nome.

Gera resumos

Cria uma daily note com tudo que foi processado, movido e conectado naquela execução.

Dicas para ir além

  • Frontmatter é poder. Defina no CLAUDE.md que toda nota precisa de tags, date e related no YAML. Isso estrutura o vault pra buscas futuras.
  • Use templates. Crie templates em templates/ para tipos de nota recorrentes (reunião, artigo, ideia). O agente vai usar como referência.
  • MOCs (Maps of Content). Peça pro agente manter MOCs — notas-índice que agrupam notas de um tema. Funciona como uma curadoria automática.
  • Sincronize com a nuvem. Use iCloud, OneDrive ou Syncthing pra acessar o vault no celular via Obsidian mobile. Jogue notas na inbox de qualquer lugar.
  • Revise o daily log. Gaste 2 minutos de manhã lendo o que o agente fez na noite anterior. Você vai descobrir conexões que não tinha visto.
