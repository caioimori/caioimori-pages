/teleport e /remote-control: sessoes cross-device de verdade

O leak apontou cedo para um fluxo de Claude Code menos preso a uma unica maquina. Hoje as docs publicas ja deixam isso claro: voce pode criar sessoes remotas na web, teleportar uma sessao web para o terminal e controlar uma sessao local pelo celular ou navegador.

Nao misture

Sao 3 fluxos diferentes

A maior fonte de confusao aqui e tratar tudo como a mesma coisa. Na pratica, `--remote`, `/teleport` e `/remote-control` resolvem problemas diferentes.

  • --remote: cria uma nova sessao na web, rodando na nuvem.
  • /teleport: puxa uma sessao web para o seu terminal local.
  • /remote-control: expoe a sua sessao local para voce continuar de outro device.
  • Mobile: funciona com claude.ai/code e com o app Claude no iOS/Android.
Web --remote

Cria uma tarefa nova na nuvem, monitoravel no navegador e no app mobile.

Handoff /teleport

Puxa a sessao web para o terminal com branch, repo e historico completos.

Controle remoto /remote-control

Conecta o celular ou browser a uma sessao local que continua rodando na sua maquina.

Versao 2.1.51+

Remote Control exige Claude Code 2.1.51 ou superior segundo a documentacao oficial.

00

Antes de tudo: nao confunda os 3 fluxos

Se voce entender essa tabela, ja evita 90% da confusao sobre sessoes cross-device no Claude Code.

claude --remote Cria uma nova sessao web no claude.ai/code. O trabalho roda na infra da Anthropic, nao na sua maquina.
/teleport Puxa uma sessao que ja esta na web de volta para o seu terminal local, com historico completo.
/remote-control Permite continuar uma sessao local do seu desktop via browser ou app mobile. A sessao continua rodando na sua maquina.

Leitura certa: --remote e cloud. /teleport e handoff da cloud para o terminal. /remote-control e espelho da sua sessao local para outro device.

01

Como funciona o fluxo com /teleport

`/teleport` entra quando voce comecou uma tarefa na web e quer trazer essa mesma sessao para o terminal local sem perder contexto. 

# criar uma nova sessao na web
claude --remote "Corrige o bug de autenticacao em src/auth/login.ts"

# depois, no terminal local, puxar a sessao de volta
/teleport

# ou usar o binario direto
claude --teleport
claude --teleport <session-id>

Onde comeca

Primeiro voce dispara uma tarefa na web com --remote. Ela aparece em claude.ai/code e no app mobile.

Como voltar para o terminal

Use /teleport, claude --teleport ou pressione t dentro de /tasks.

O que e carregado

O Claude verifica repo correto, busca a branch remota, faz checkout e carrega o historico completo da conversa.

02

Requisitos reais do /teleport

A doc oficial lista quatro checagens objetivas antes de uma sessao ser teleportada para o terminal.

Checklist

  • Git limpo: sem changes locais nao commitadas, ou voce vai ter que stashear.
  • Repo correto: o checkout local precisa ser do mesmo repositorio, nao um fork.
  • Branch publicada: a branch da sessao web precisa existir no remoto.
  • Mesma conta: voce precisa estar autenticado no mesmo claude.ai da sessao web.

A nuance importante

  • O handoff e one-way nesse sentido: voce puxa uma sessao web para o terminal.
  • Voce nao "empurra" uma sessao local existente para a web com teleport.
  • Para iniciar algo na web, o fluxo certo e claude --remote.
  • Teleport resolve continuidade, nao espelhamento ao vivo.
03

Como funciona o /remote-control

`/remote-control` e outro jogo: aqui voce nao move a sessao para a nuvem. Voce abre uma janela remota para uma sessao que continua rodando localmente na sua maquina. 

# modo servidor dedicado
claude remote-control

# sessao interativa local, mas acessivel remotamente
claude --remote-control
claude --remote-control "Meu Projeto"

# ativar dentro de uma sessao que ja esta rodando
/remote-control
/remote-control Meu Projeto

O que fica local

Filesystem, MCP servers, tools e configuracao continuam na sua maquina. O web/mobile vira so a janela dessa sessao.

Como conectar do celular

O terminal mostra uma URL de sessao e um QR code. Da para abrir no claude.ai/code ou direto no app Claude.

Como usar junto

Voce pode digitar do terminal, do browser e do celular na mesma sessao. Tudo fica sincronizado em tempo real.

04

Requisitos e limites do Remote Control

Essa parte importa porque o comportamento muda bastante dependendo do plano e da forma de login.

Versao minima Claude Code v2.1.51+.
Planos Disponivel em Pro, Max, Team e Enterprise. Em Team/Enterprise, admin precisa ligar o toggle.
Auth Precisa de login via claude.ai. API key sozinha nao serve para Remote Control.
Workspace trust Voce precisa abrir o projeto no Claude Code ao menos uma vez para aceitar o trust dialog.
Terminal aberto Se voce fechar o terminal ou matar o processo local, a sessao acaba.
Onde a sessao roda Na sua maquina, com trafego saindo por HTTPS via API da Anthropic. Nao abre porta inbound local.

A diferenca mais importante: no Remote Control, a Anthropic nao esta executando seu repo em infra propria. Ela so intermedeia a conexao com a sessao que continua viva no seu computador.

05

Quando usar cada um

Se voce escolher o fluxo errado, o problema nao e a feature. E o modelo mental.

Use --remote + /teleport quando

  • Voce quer disparar uma tarefa na nuvem e seguir com a vida.
  • Quer monitorar progresso pelo celular enquanto o trabalho roda no cloud.
  • Quer voltar para o terminal depois para revisar, ajustar ou finalizar localmente.
  • Quer rodar varias tasks paralelas independentes em sessoes web separadas.

Use /remote-control quando

  • Voce ja esta com uma sessao local boa e quer continuar de outro device.
  • Precisa do seu filesystem, MCPs, worktree e configuracao exatamente como estao na sua maquina.
  • Quer alternar entre desktop, browser e mobile sem trocar o lugar onde a sessao executa.
  • Quer trabalhar de forma sincrona em uma mesma conversa viva entre superficies.
06

Resumo rapido

Se voce precisar explicar isso em 20 segundos, usa esta tabela.

O que faz --remote Cria uma nova sessao web no claude.ai/code, rodando na nuvem.
O que faz /teleport Puxa uma sessao web existente para o terminal local, com branch e historico completos.
O que faz /remote-control Continua uma sessao local pelo browser ou celular, sem mover a execucao para a nuvem.
Qual usa mobile melhor Os dois podem envolver mobile, mas em modelos diferentes: monitorar/steer web sessions ou controlar sua sessao local.
Onde as pessoas mais erram Confundindo handoff de sessao com espelhamento remoto ao vivo.
