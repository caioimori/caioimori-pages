Resumo em 30 segundos

Se voce quer o atalho sem ler o guia inteiro, siga esta ordem.

  1. macOS

    Abra o Terminal e rode curl -fsSL https://claude.ai/install.sh | bash.

  2. Windows

    Instale Git for Windows primeiro. Depois, no PowerShell, rode irm https://claude.ai/install.ps1 | iex.

  3. Verifique

    Rode claude --version e claude doctor.

  4. Entre no projeto

    Va para a pasta do repo e execute claude. No primeiro uso, o login abre no navegador.

01

O que voce precisa antes de instalar

Esta e a parte que evita 90% dos erros de setup.

Requisitos do sistema

macOS 13+, Windows 10 1809+ ou Windows Server 2019+, 4 GB+ de RAM e conexao com a internet.

Shell suportado

Bash, Zsh, PowerShell e CMD estao cobertos oficialmente.

Conta elegivel

Claude Code exige Pro, Max, Team, Enterprise ou Console. O plano gratuito do Claude.ai nao entra.

Windows nativo

  • Git for Windows e obrigatorio porque Claude Code usa Git Bash internamente.
  • Voce pode abrir claude a partir de PowerShell, CMD ou Git Bash.
  • Voce nao precisa rodar PowerShell como administrador.

Node.js

  • Nao e necessario para o instalador nativo.
  • So use Node.js 18+ se voce realmente precisar do metodo npm por compatibilidade.
  • A propria documentacao diz que npm esta descontinuado.
02

Instalacao no macOS

No Mac, o caminho mais simples continua sendo o instalador nativo. Homebrew entra como alternativa boa para quem centraliza tudo ali.

Opcao recomendada: instalador nativo

curl -fsSL https://claude.ai/install.sh | bash

Esse metodo instala o binario nativo e recebe atualizacoes automaticas em segundo plano.

Alternativa: Homebrew

brew install --cask claude-code

Funciona bem, mas nao atualiza sozinho. Depois voce precisa rodar brew upgrade claude-code.

  1. Abra o Terminal

    Use o Terminal padrao do macOS, iTerm ou outro shell compativel.

  2. Execute a instalacao

    Rode o comando nativo acima. Se voce prefere Homebrew, use o comando alternativo.

  3. Cheque se o comando entrou no PATH

    Rode claude --version. Se o comando responder, a instalacao entrou.

  4. Entre em um projeto e abra o Claude

    Va para seu repo e rode:

    cd seu-projeto claude

Se voce quiser menos risco de regressao: o instalador nativo aceita o canal stable.

curl -fsSL https://claude.ai/install.sh | bash -s stable
03

Instalacao no Windows

Aqui esta a principal pegadinha: no Windows nativo, Git for Windows vem antes do Claude Code.

Passo zero

Instale Git for Windows e marque a opcao Add to PATH.

Shell certo

Se estiver no PowerShell, use o comando irm. Se estiver no CMD, use o instalador .cmd.

Alternativa oficial

Se voce prefere pacote do Windows, o caminho oficial alternativo e o WinGet.

Opcao recomendada: PowerShell

irm https://claude.ai/install.ps1 | iex

Este e o comando oficial para Windows PowerShell.

Alternativas oficiais

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
winget install Anthropic.ClaudeCode

CMD usa o instalador .cmd. WinGet tambem funciona, mas nao atualiza sozinho.

  1. Instale o Git for Windows primeiro

    Sem isso, o Claude Code nativo no Windows falha com o erro Claude Code on Windows requires git-bash.

  2. Abra o shell certo

    PowerShell usa irm ... | iex. CMD usa o instalador install.cmd.

  3. Execute a instalacao

    Rode um dos comandos acima. O nativo via PowerShell continua sendo o principal.

  4. Valide o binario

    Rode claude --version e depois claude doctor.

  5. Abra seu projeto

    Entre na pasta do repo e rode claude.

Se o Git ja esta instalado e mesmo assim falha: a documentacao oficial orienta configurar manualmente o caminho do Git Bash no settings.json.

{ "env": { "CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe" } }
04

Primeira verificacao e primeiro login

Depois da instalacao, o objetivo e confirmar tres coisas: o binario existe, o PATH esta certo e a autenticacao abre normalmente.

claude --version claude doctor cd seu-projeto claude

O que voce deve ver

  • claude --version responde com a versao instalada.
  • claude doctor faz a checagem mais detalhada do ambiente.
  • No primeiro claude, o navegador abre para login.

Quem pode entrar

  • Claude Pro
  • Claude Max
  • Teams, Enterprise ou Console
  • Ou provedores de terceiros como Bedrock, Vertex AI e Microsoft Foundry
05

Como atualizar depois

Aqui muita gente confunde comportamento do instalador nativo com Homebrew e WinGet.

Instalacao nativa

  • Atualiza automaticamente em segundo plano.
  • Se quiser forcar a atualizacao, rode claude update.
  • Voce pode usar o canal stable para fugir das ultimas novidades imediatamente.

Homebrew e WinGet

brew upgrade claude-code winget upgrade Anthropic.ClaudeCode

Esses dois metodos nao fazem update automatico.

06

Erros comuns e como sair deles rapido

Aqui estao os erros que mais aparecem na documentacao oficial de troubleshooting.

"&&" invalido no Windows

Voce esta no PowerShell usando o comando de CMD. Troque para irm https://claude.ai/install.ps1 | iex.

"irm" nao reconhecido

Voce esta no CMD, nao no PowerShell. Abra PowerShell ou use o instalador .cmd.

"claude" nao e reconhecido

O binario nao entrou no PATH. No macOS ele fica em ~/.local/bin; no Windows, em %USERPROFILE%\\.local\\bin.

Ajuste rapido de PATH no macOS

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc claude --version

Ajuste rapido de PATH no PowerShell

$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User') [Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\\.local\\bin", 'User')

Depois feche e abra o terminal e rode claude --version.

Script retornando HTML

Normalmente isso aponta para problema temporario de rede, bloqueio regional, proxy corporativo ou indisponibilidade de rota. Tente Homebrew no macOS ou WinGet no Windows.

Windows abre o app Desktop em vez do CLI

Uma versao antiga do Claude Desktop pode tomar prioridade no PATH. A correcao oficial e atualizar o app Desktop.
